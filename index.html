Doodle Polls

A lightweight, GitHub-hosted scheduling poll — no server required.

Open Setup

How it works

  1. Open setup.html and fill in your GitHub credentials once (stored in root config.txt).
  2. Use the New Doodle section in setup to generate a subfolder config and push it to GitHub.
  3. Copy the template/ folder in the repo to the new subfolder name (e.g. team-meeting/).
  4. Share the poll URL with participants:
    https://YOUR_USERNAME.github.io/YOUR_REPO/team-meeting/
  5. View results at:
    https://YOUR_USERNAME.github.io/YOUR_REPO/team-meeting/results.html

Creating a new doodle

The fastest way — just copy the template folder:

  1. Duplicate the template/ folder and rename it (e.g. birthday-party/).
  2. Edit birthday-party/config.txt — set title, organizer, access word, and time slots.
  3. Commit and push. GitHub Pages serves it automatically.

Or use setup.html to create and push the config via the GitHub API without touching git.

File structure

config.txt — global GitHub owner, repo, branch, token

template/config.txt — per-doodle title, organizer, access word, time slots
template/index.html — voting page (auto-detects folder from URL)
template/results.html — results page
template/responses/ — responses saved here as .txt files
